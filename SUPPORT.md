# Support

Repo2Prompts is maintained as a public self-hosted project. Support is
best-effort.

## When to open a GitHub issue

Open a public issue for:

- reproducible bugs
- documentation problems
- feature requests
- UX or developer-experience improvements

Please include:

- the repo commit or branch you tested
- the commands you ran
- logs or screenshots when they materially help

## When not to open a public issue

Do not use public issues for:

- secrets or credential exposure
- security vulnerabilities
- sensitive production data

Use `SECURITY.md` for vulnerability disclosure instead.

## Self-hosted expectations

Repo2Prompts supports self-hosted setups, but environment-specific problems may
depend on your model gateway, database, hosting layer, or local machine.

For the fastest triage, include:

- Node and package manager versions
- model provider or OpenAI-compatible gateway details
- whether Supabase durable storage is enabled
- exact failing command and output
